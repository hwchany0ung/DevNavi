## Executive Summary

| 관점 | 내용 |
|------|------|
| **Problem** | 사용자가 로드맵 태스크("Docker 네트워킹 설정하기")를 보고 **무엇을 해야 하는지, 어떻게 시작해야 하는지 모르는 상태에서 이탈**한다. 현재 DevNavi는 로드맵을 생성하지만 개별 태스크 수행을 돕는 메커니즘이 없어, 사용자는 외부 검색(구글, ChatGPT)에 의존한다. |
| **Solution** | 각 태스크 항목 옆에 **인라인 AI Q&A 버튼**을 추가하여, 해당 태스크의 맥락(직군, 월차, 주차, 카테고리, 선행 학습 내용)을 자동 주입한 Claude AI 대화를 제공한다. 사용자는 "이거 어떻게 하나요?", "예시 보여줘" 등을 태스크 화면 안에서 즉시 질문할 수 있다. |
| **기능 UX 효과** | 로드맵 페이지 내 **컨텍스트 유지 학습** 실현 -- 외부 이탈 없이 태스크 이해 -> 실행 -> 완료 루프를 단일 화면에서 완성. 학습 몰입도 및 태스크 완료율 향상. |
| **Core Value** | **"AI가 만든 로드맵을 AI가 실행까지 도와준다"** -- 계획 생성(Plan)에서 실행 지원(Do)으로 가치 체인 확장. IT 취업 준비생의 자기주도 학습 완결성 달성. |

## Part 1: Discovery Analysis (pm-discovery)

### 1.1 Five-Step Discovery Chain

#### Step 1 -- Brainstorm (기회 발견)

| # | 기회 영역 | 설명 |
|---|----------|------|
| O1 | 태스크 수행 가이드 부재 | "Docker 네트워킹 설정"이라는 태스크만 있고, 무엇부터 해야 하는지 안내가 없음 |
| O2 | 외부 검색 이탈 | 태스크 내용을 모르면 구글/ChatGPT로 이동 -- DevNavi 체류시간 감소 |
| O3 | 맥락 단절된 학습 | 외부 검색 결과는 사용자의 직군/수준/진행상황을 모름 -- 일반적 답변만 제공 |
| O4 | 체크리스트 완료율 정체 | 어려운 태스크에서 멈추면 이후 태스크도 미완료로 누적 |
| O5 | 재방문 동기 부족 | 도움 없이 혼자 해결해야 하므로 로드맵을 방치하게 됨 |

#### Step 2 -- Assumptions (핵심 가정)

| # | 가정 | Impact (1-5) | Risk (1-5) | Score |
|---|------|:---:|:---:|:---:|
| A1 | 사용자가 태스크 내용을 모를 때 DevNavi 내에서 도움을 원한다 | 5 | 2 | 10 |
| A2 | AI가 태스크 맥락(직군, 주차, 선행학습)을 알면 더 유용한 답변을 준다 | 5 | 2 | 10 |
| A3 | 인라인 Q&A가 태스크 완료율을 20% 이상 올린다 | 4 | 3 | 12 |
| A4 | 사용자는 긴 대화보다 1-2턴의 짧은 Q&A를 선호한다 | 4 | 3 | 12 |
| A5 | API 비용이 기능 유지에 부담이 되지 않는다 | 3 | 4 | 12 |

#### Step 3 -- Prioritize (우선순위)

**Top-3 Riskiest Assumptions** (Score 기준 내림차순):
1. **A3** (완료율 20% 향상) -- 실제 사용 데이터 필요
2. **A4** (짧은 Q&A 선호) -- 사용 패턴 추적으로 검증
3. **A5** (API 비용 지속 가능성) -- 호출량 x 단가 시뮬레이션 필요

#### Step 4 -- Experiments (검증 실험)

| 가정 | 실험 | 성공 기준 |
|------|------|----------|
| A3 | Q&A 버튼 사용자 50명 A/B 테스트 (2주) | Q&A 사용 그룹 태스크 완료율 >= 대조군 + 15% |
| A4 | Q&A 대화 턴 수 추적 | 80% 세션이 3턴 이내 종료 |
| A5 | Haiku 모델 기반 비용 시뮬레이션 | 월 활성 사용자 1,000명 기준 API 비용 < $50 |

#### Step 5 -- Opportunity Solution Tree (OST)

```
[Outcome] 태스크 완료율 30% -> 60% 향상
├── [O1] 태스크 수행 가이드 부재
│   ├── [S1] 인라인 AI Q&A (핵심 기능)
│   │   ├── 태스크별 "물음표" 버튼
│   │   ├── 맥락 자동 주입 (직군/월/주/카테고리)
│   │   └── Haiku 모델 빠른 응답
│   └── [S2] 태스크 미리보기 tooltip
│       └── 태스크 hover 시 간단 설명 표시
├── [O2] 외부 검색 이탈 방지
│   ├── [S1] 인라인 AI Q&A (재활용)
│   └── [S3] 추천 리소스 링크 자동 생성
├── [O3] 맥락 단절 학습
│   └── [S1] 인라인 AI Q&A -- 선행 태스크 완료 정보 포함
├── [O4] 체크리스트 완료율 정체
│   └── [S4] 미완료 태스크 리마인더 + Q&A 유도
└── [O5] 재방문 동기 부족
    └── [S5] "오늘의 학습 도우미" 푸시알림
```

**핵심 솔루션: S1 (인라인 AI Q&A)** -- 4개 기회(O1~O4)를 동시에 해결하는 레버리지 포인트.

## Part 2: Strategy Analysis (pm-strategy)

### 2.1 JTBD 6-Part Value Proposition

| Part | 내용 |
|------|------|
| **1. When** | 로드맵의 주차별 태스크를 확인하고 실행하려 할 때 |
| **2. I want to** | 이 태스크가 구체적으로 무엇을 의미하는지, 어떻게 시작/완료하는지 즉시 알고 싶다 |
| **3. So I can** | 외부 검색 없이 태스크를 이해하고 완료하여 학습 진도를 유지할 수 있다 |
| **4. Job Performer** | IT 직군 취업을 준비하는 비전공자/초보 개발자 (20대 중후반) |
| **5. Functional Job** | 태스크 내용 이해, 실행 방법 파악, 예시 코드/자료 확인, 완료 여부 판단 |
| **6. Emotional Job** | "나도 할 수 있다"는 자신감 유지, 학습 좌절 방지, 혼자 공부하는 외로움 해소 |

### 2.2 Lean Canvas

| 구분 | 내용 |
|------|------|
| **Problem** | (1) 태스크가 무엇인지 모름 (2) 외부 검색 결과가 내 수준/맥락과 맞지 않음 (3) 혼자 공부 시 막히면 포기 |
| **Customer Segments** | IT 취업 준비생 (비전공 전환자, 부트캠프 수강생, 독학 개발자) |
| **Unique Value Proposition** | "AI가 만든 로드맵을, 같은 AI가 태스크 단위로 코칭해준다" -- 계획-실행 일체형 학습 |
| **Solution** | 각 태스크 옆 AI Q&A 버튼 -> 직군/수준/진행도 맥락이 주입된 Claude AI 답변 |
| **Channels** | DevNavi 웹앱 내 로드맵 페이지 (인라인 통합) |
| **Revenue Streams** | (1) 프리미엄 구독 차별 기능 (무료: 3회/일, 프리미엄: 무제한) (2) 향후 기업 B2B 라이선스 |
| **Cost Structure** | Claude Haiku API 호출 비용 (주요), 서버/인프라 비용 (경미) |
| **Key Metrics** | Q&A 사용률, 태스크 완료율 변화, 세션 체류시간, 일별 재방문율 |
| **Unfair Advantage** | (1) 로드맵 생성 시점의 사용자 맥락 데이터 보유 (2) 9개 직군별 커리큘럼 최적화 프롬프트 (3) 태스크 진행도 실시간 추적 |

### 2.3 SWOT Analysis

| | 긍정적 | 부정적 |
|---|--------|--------|
| **내부** | **S** -- 이미 Claude API 통합 완료, 9개 직군 프롬프트 최적화 경험, 태스크 데이터 구조화 완료 | **W** -- 단일 개발자 프로젝트 (속도 제한), API 비용 통제 경험 부족, 사용자 기반 소규모 |
| **외부** | **O** -- AI 튜터 시장 급성장 (Khanmigo 성공 사례), 한국 에듀테크 10조 시장, IT 취업 수요 지속 | **T** -- ChatGPT/Gemini 범용 AI가 Q&A 대체 가능, 인프런/코드잇의 AI 기능 강화, API 가격 변동 |

**SO 전략**: 기존 프롬프트 역량 + AI 튜터 시장 성장 -> 직군 특화 맥락 Q&A로 차별화
**WT 대응**: API 비용 최적화 (Haiku 우선, 캐싱), 범용 AI 대비 "학습 맥락 유지" 가치 강조

### 2.4 Porter's Five Forces (간략)

| Force | 수준 | 근거 |
|-------|------|------|
| 신규 진입 위협 | 높음 | AI API만 있으면 유사 서비스 구축 가능 |
| 대체재 위협 | 높음 | ChatGPT, Gemini로 직접 질문 가능 |
| 공급자 교섭력 | 중간 | Anthropic API 가격 정책 의존 (다만 다모델 전환 가능) |
| 구매자 교섭력 | 높음 | 무료 학습 리소스 풍부, 전환 비용 낮음 |
| 경쟁 강도 | 중간 | 직군 특화 AI 로드맵 + Q&A 결합 서비스는 희소 |

**전략적 시사점**: 대체재(범용 AI) 위협이 가장 큼 -> "맥락 인지 + 로드맵 연계"로 방어

```
[1. 인지]          [2. 온보딩]        [3. 로드맵 확인]     [4. 태스크 실행]     [5. Q&A 활용]       [6. 완료/재방문]
탐색/광고 유입  ->  직군/수준 입력  ->  월차별 체크리스트  ->  태스크 클릭      ->  AI에게 질문     ->  체크 완료 + 재방문
                     |                   |                    |                   |                    |
감정: 기대감       감정: 설렘          감정: 압도감          감정: 당혹감       감정: 안도감          감정: 성취감
                                       "양이 많네..."       "이게 뭐지?"      "아, 이런 뜻이구나"   "오늘 3개 완료!"
                                                              |                   |
                                                         [현재 이탈 지점]    [Q&A가 해결하는 구간]
```

**핵심 인사이트**: 4단계(태스크 실행)에서 5단계(Q&A 활용)로의 전환이 이 기능의 핵심 가치. 현재 4단계에서 외부 이탈이 발생하며, AI Q&A는 이 이탈을 차단하고 5단계로 즉시 연결한다.

## Part 5: Requirements & Specifications

### 5.1 Feature Requirements

#### FR-01: TaskItem Q&A 버튼

- 각 `TaskItem` 컴포넌트에 물음표(?) 아이콘 버튼 추가
- 클릭 시 해당 태스크에 대한 Q&A 패널/모달 오픈
- 비로그인 사용자: Q&A 버튼 클릭 시 로그인 유도 모달

#### FR-02: Q&A 패널 UI

- 태스크 아이템 아래에 슬라이드-다운 인라인 패널 또는 사이드 패널
- 입력: 자유 텍스트 질문 (최대 500자)
- 출력: AI 답변 SSE 스트리밍 (기존 stream_teaser 패턴 재활용)
- 대화 이력: 세션 내 최대 5턴 유지 (localStorage)
- Quick Actions: "이게 뭐예요?", "예시 보여줘", "핵심만 요약" 프리셋 버튼 3개

#### FR-03: 맥락 자동 주입

- 시스템 프롬프트에 자동 포함:
  - `role`: 사용자의 목표 직군 (예: backend)
  - `level`: 사용자 수준 (예: beginner)
  - `month/week`: 현재 월차/주차
  - `task.content`: 태스크 내용 (예: "Docker 네트워킹 설정하기")
  - `task.category`: learn/project/cert
  - `completed_tasks`: 같은 주차 내 완료된 태스크 목록 (선행학습 맥락)
  - `month.theme`: 월차 테마 (예: "컨테이너화 & CI/CD")

#### FR-04: 사용량 제한

- 무료 사용자: 일 3회 Q&A 세션 (1세션 = 1태스크 대화)
- 프리미엄 사용자: 무제한
- 제한 도달 시: "오늘의 AI 도움 횟수를 모두 사용했어요. 프리미엄으로 업그레이드하면 무제한으로 질문할 수 있어요!" + 업그레이드 CTA

#### FR-05: API 엔드포인트

- `POST /roadmap/{roadmap_id}/task-qa` -- 태스크별 Q&A 스트리밍
- Request Body:
  ```json
  {
    "task_id": "1-1-0",
    "question": "이 태스크 어떻게 하나요?",
    "conversation_history": [
      {"role": "user", "content": "..."},
      {"role": "assistant", "content": "..."}
    ]
  }
  ```
- Response: SSE stream (기존 `stream_teaser` 패턴)
- Rate limit: 30/hour (per user)
- Model: Claude Haiku (빠른 응답 + 저비용)

#### FR-06: 프롬프트 엔지니어링

- 시스템 프롬프트 구조:
  ```
  당신은 DevNavi의 AI 학습 도우미입니다.
  사용자는 {role} 직군 취업을 준비하는 {level} 수준의 학습자입니다.
  현재 {month}월차 "{month_theme}" 단원의 {week}주차를 학습 중입니다.
  
  [현재 태스크]
  - 내용: {task.content}
  - 카테고리: {task.category}
  
  [이미 완료한 태스크]
  {completed_tasks_list}
  
  [지침]
  - 사용자의 수준에 맞게 쉽게 설명하세요
  - 한국어로 답변하세요
  - 필요하면 코드 예시를 포함하세요
  - 답변은 간결하게 (300자 이내 텍스트 + 코드 별도)
  - 직접 답을 주되, "이것도 해보세요" 형태의 다음 학습 제안을 1개 포함하세요
  ```

### A. 기술 아키텍처 개요

```
[Frontend]                              [Backend]                          [External]
TaskItem.jsx                            /roadmap/{id}/task-qa              Claude Haiku API
  ├── QAButton (물음표 아이콘)           ├── auth: require_user
  ├── QAPanel (인라인 슬라이드다운)      ├── rate_limit: 30/hour
```
